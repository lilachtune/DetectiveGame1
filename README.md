# 🕵️ Детективная игра — расследуй убийство в Godot Engine

**Жанр:** Тыкни-ищи (Point & Click) / Визуальная новелла-детектив  
**Движок:** Godot 4.2+ 
**Разрешение:** 1920 × 1080 (Full HD)  
**Язык:** Русский 

---

## 📖 О чём игра?

Вы — **следователь**, прибывший на место загадочного убийства в роскошном особняке. Ваша задача:

1. **Осмотреть 15 локаций** — от парадного холла до тёмного чердака.
2. **Найти улики** — окурки, визитки, царапины на замках, остатки ужина и многое другое.
3. **Поговорить с 10 подозреваемыми** — каждый имеет мотив, алиби и свои секреты.
4. **Вести дневник** — туда автоматически попадают все открытые локации, найденные улики и записанные диалоги.
5. **Сдать финальный тест** — ответить на три вопроса: *кто убил? каким способом? зачем?*

Игра **автосохраняется** при каждом переходе между локациями и при выходе. Ничего не потеряется.

## 🎮 Как скачать и запустить игру

### Способ 1: Открыть проект в редакторе (для модов и разработки)

> ⚠️ Нужен установленный **Godot 4.2+** (скачать: [godotengine.org](https://godotengine.org/download/))

1. Скачай репозиторий (архив ZIP или `git clone`).
  
   ```bash
   git clone https://github.com/lilachtune/DetectiveGame1.git
   ```

2. Открой Godot → нажми **Import** → выбери папку с проектом (ту, где лежит `project.godot`).

3. **Сразу готово!** В проекте уже настроены:
   - ✅ Все 4 автозагрузчика (Autoload) — GameManager, SaveManager, AudioManager, DiaryManager
   - ✅ Аудиошины (Master → Music → SFX)
   - ✅ Главная сцена (`MainMenu.tscn`)
   - ✅ Кастомная тема оформления

4. Нажми **F5** (или кнопку Play ▶) — игра запустится.

### Способ 2: Собрать свою версию (экспорт)

1. В Godot: **Project → Export**.
2. Добавь шаблон для своей платформы (скачиваются из меню Export).
3. Настрой параметры → **Export PCK/ZIP**.
4. Готово — можно раздавать друзьям.

## 🏗️ Как всё устроено внутри (обзор архитектуры)

Проект построен на **Autoload-синглтонах** и наследовании сцен. Это значит, что вся «глобальная» логика видна отовсюду, а каждая локация и персонаж собираются из кубиков.

```
autoload/           ← «Мозги» игры — глобальные менеджеры
├── GameManager     ← состояние игры: меню / играем / диалог / пауза / конец
├── SaveManager     ← автосохранение, загрузка, удаление сейва
├── AudioManager    ← музыка и звуковые эффекты 
└── DiaryManager    ← дневник: персонажи, улики, локации, диалоги

scripts/            ← Логика сцен и интерфейса
├── LocationBase    ← базовый класс для всех 15 локаций
├── CharacterBase   ← базовый класс для персонажей
├── DialogueBox     ← окошко диалогов с портретом и печатающимся текстом
├── Diary           ← сам дневник (3 вкладки + детальный просмотр)
├── Settings        ← настройки громкости
├── PauseMenu       ← меню паузы (Escape)
└── EndingTest      ← финальный тест: убийца, способ, мотив

scenes/locations/   ← 15 файлов .tscn — каждая локация со своим фоном
resources/          ← вспомогательные ресурсы (DialogueData — конструктор диалогов)

assets/             ← всё «тяжёлое»:
├── characters/     ← PNG-спрайты и портреты персонажей
├── locations/      ← PNG-фоны локаций
└── ui/             ← иконки и прочее
```

### Как переключаются локации

Каждая локация — это отдельная `.tscn`. При переходе игра:
1. Сохраняется автоматически.
2. Загружает новую сцену.
3. Регистрирует локацию как «открытую» в дневнике.
4. Ставит фоновую музыку (если есть в экспорт-переменной локации).

### Как работают диалоги

В скрипте локации или персонажа создаётся словарь диалога с помощью `DialogueData.make_dialogue(...)`.  
Он передаётся в `start_dialogue(...)` → появляется окно с портретом говорящего, текст печатается по буквам.  
После завершения диалог **автоматически** попадает в дневник на вкладку персонажа.

### Как работает дневник

Кнопка с иконкой блокнота → открывается наложение с интерфейсом.  
Внутри **три вкладки**:

| Вкладка | Что показывает |
|---------|----------------|
| 🗺️ Локации | Какие места открыты, какое заблокировано, какое текущее |
| 🕵️ Персонажи | Список тех, с кем уже говорили. Клик — подробности: фото, инфа, мотив, алиби, допросы |
| 🔍 Улики | Все найденные предметы. Клик — детальное описание и обстоятельства |

Когда сюжет пройден до конца — на вкладке персонажа появляется кнопка **«Тест»**.

### Как работает финальный тест

После того, как `GameManager.complete_story()` разблокирует кнопку, игрок выбирает из выпадающих списков:
- **Кто убийца** (только те персонажи, которых открыли)
- **Способ убийства**
- **Мотив**

Если всё верно — «Дело раскрыто!», можно делать титры. Если нет — скажет, что именно не так.

## 💾 Где лежит сохранение

Путь: `user://savegame.save`

**Windows:** `%APPDATA%\Godot\app_userdata\Detective Game\savegame.save`  
**Linux:** `~/.local/share/godot/app_userdata/Detective Game/savegame.save`  
**macOS:** `~/Library/Application Support/Godot/app_userdata/Detective Game/savegame.save`

Сохранение — обычный JSON. Можно открыть блокнотом и редактировать (но не ломай).
